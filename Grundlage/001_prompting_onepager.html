A) Was bedeutet „effizientes und strukturiertes Prompting“?

Kurz gesagt: Die KI erhält nicht nur eine vage Bitte, sondern einen klar strukturierten Arbeitsauftrag mit Ziel, Kontext, Input, Regeln und gewünschtem Ausgabeformat.

Effizient

Weniger Nacharbeit, schneller brauchbare Ergebnisse.

Strukturiert

Task, Input und Regeln sind sauber getrennt.

Reproduzierbar

Vorlagen funktionieren auch im Team.

Praxistauglich

Direkt nutzbar für Berichte, Anfragen, Prüfungen.

Vorteile

  • Klarere Ergebnisse beim ersten Versuch
  • Bessere Steuerung von Länge, Ton und Format
  • Gut für komplexe Fach- und Prüfungskontexte
  • Leicht wiederverwendbare Team-Vorlagen

Nachteile / Grenzen

  • Etwas mehr Aufwand beim initialen Prompt-Design
  • Zu viel Struktur kann bei sehr einfachen Anfragen überladen sein
  • Fachprüfung bleibt nötig (insb. bei Recht/Regulatorik)

Schlechter vs. guter Prompt (Einstieg)

Schlechter Prompt (zu vage)

Fass das zusammen und mach es professionell. Achte auf die wichtigen Punkte und schreibe es so, dass es für alle passt.

Problem: Keine Zielgruppe, kein Format, keine Länge, keine Regeln.

Guter Prompt (strukturiert)

# Aufgabe
Fasse den Text für die Abteilungsleitung zusammen.

## Ziel
- max. 120 Wörter
- entscheidungsorientiert
- Risiken und offene Punkte nennen

## Ausgabeformat
1. Kernaussage
2. Risiken
3. Nächste Schritte

## Input
<<<TEXT>>>
...
<<<END>>>

Mehr Struktur = schneller besserer Output = weniger Iterationen.

Merksatz: Erst Task, dann Ziel/Regeln, dann Input, dann Output-Format.

C) Wie setzt man strukturiertes Prompting in der Praxis ein? (Das „Wie“)

Eine einfache Arbeitsroutine für den Alltag — für den sofortigen Start.

Task benennen

z. B. zusammenfassen, prüfen, entwerfen, vergleichen.

Zielgruppe & Ziel definieren

Wer liest das Ergebnis? Was muss am Ende vorliegen?

Input abgrenzen

Immer Delimiter nutzen: <<<INPUT>>> ... <<<END>>>

Output-Format zeigen

Überschriften, Tabellen, Felder, Länge und Pflichtpunkte vorgeben.

Regeln ergänzen

Keine neuen Fakten, fehlende Infos markieren, Fakten/Interpretation trennen.

Gezielt iterieren

Nur 1–2 Elemente je Runde ändern (Format, Zielgruppe, Detailtiefe).

Vorlage speichern

Erfolgreiche Prompts als Template sichern und wiederverwenden.

Praxisformel: Task → Ziel → Kontext → Input (Delimiter) → Output-Format → Regeln → Iteration

D) Cluster-Bündelung der Use Cases in drei Hauptgruppen

Die Bündelung macht den Transfer leichter: Wiederkehrende Prompt-Muster werden sofort erkennbar.

1) Zusammenfassen & Aufbereiten

  • Meetings & Protokolle
  • Wissenschaftliche Publikationen
  • Experteninput für Zielgruppen

Quick Wins, idealer Einstieg, hoher Aha-Effekt.

2) Erstellung & Überarbeitung fachlicher Texte

  • Antwortschreiben / Anfragen
  • Bürgeranfragen (allgemeinverständlich)
  • Arbeitsanweisungen / Projektdokumentation

Direkt im Arbeitsalltag einsetzbar und teamfähig.

3) Prüfung / Bewertung / Guideline-Abgleich

  • Bewertung nach Kriterien
  • Gap-Analyse mit Leitlinien/DIN

Advanced, zeigt den Mehrwert von XML-/Schema-Logik.

Welche Formate wann? (sehr praktisch, kurz)

  • Markdown → Zusammenfassungen, Protokolle, Textentwürfe, Vorlagen
  • XML → Regeln, Kriterien, Pflichtfelder, hierarchische Anforderungen
  • Delimiter → saubere Trennung von Input und Instruktion
  • Tabellen → Bewertung, Vergleich, To-dos, Gap-Analyse
  • Flow/UML-ähnlich → Prozesse und Rollenübergaben

Bonus: Gemeinsame Meta-Vorlage (für fast alle Fälle)

Format: Markdown + Delimiter • universell anpassbar
# Aufgabe
[Was soll erstellt / geprüft / transformiert werden?]

## Ziel
- [Was ist das gewünschte Ergebnis?]
- [Für wen?]
- [Woran erkennt man gute Qualität?]

## Kontext
- [Fachlicher Hintergrund]
- [Rahmenbedingungen]
- [Was darf nicht passieren?]

## Input
<<<INPUT>>>
[Material einfügen]
<<<END>>>

## Ausgabeformat (bitte exakt)
- [Überschriften]
- [Tabellen]
- [Länge]
- [Pflichtfelder]

## Regeln
- Nur bereitgestellte Informationen verwenden
- Fehlende Informationen markieren
- Fakten und Interpretation trennen
- [weitere spezifische Regeln]

Zusatz: Weitere Use Cases für spezielle Anforderungen

Diese Anwendungsfälle sind in der Praxis ebenfalls sehr wertvoll, erfordern aber oft spezifischeres Vorwissen oder eigene Tools.

  • Teasertexte / Website-Texte → nützlich für Kommunikation & Marketing
  • Ideensammlung / Brainstorming → gut für kreative Phasen, benötigt jedoch weniger rigide Struktur
  • Folienerstellung aus Publikationen → stark auf das finale Output-Format (z.B. VBA oder Markdown für Slides) bezogen
  • Code-Generierung (Python/R/etc.) → erfordert fortgeschrittenes Prompting bezüglich Syntax und Bibliotheken
  • CustomGPT-Erstellung (Systemprompts) → die absolute Königsdisziplin zur Automatisierung fester Arbeitsabläufe inkl. Datenanbindung
